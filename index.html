Project Overview

Project Name: Social Media Feed Dashboard

Description: This project involves building a dashboard for a simulated social media platform. Users can create posts, comment, and like posts, with data displayed in real-time. The backend is a GraphQL API built in Go, and the frontend is a React app that interacts with the backend. Docker will be used to containerize both services, making it easy to deploy.

Project Architecture

1. Backend (Go + GraphQL)

  • GraphQL API: A Go-based GraphQL API will handle CRUD operations for posts, comments, and likes.
  • Database: Use PostgreSQL to store user posts, comments, and likes.
  • Authentication: Implement basic JWT-based authentication to restrict certain actions, like creating and liking posts.

2. Frontend (React)

  • React Dashboard: A simple React app that shows posts, allows users to add new posts, comment on posts, and like them.
  • GraphQL Integration: Use Apollo Client to fetch and mutate data through the GraphQL API.
  • Real-Time Updates: Use subscriptions to enable real-time updates on the dashboard (e.g., live updating comments or like counts).

3. Docker Setup

  • Dockerize: Create Dockerfiles for the Go backend and React frontend, and use Docker Compose to orchestrate services, including PostgreSQL.

Project Breakdown

1. Backend Setup with GraphQL in Go (10 hours)

Set up a Go project with a basic GraphQL schema, implement CRUD functionality, and connect to PostgreSQL.

2. Frontend with React and GraphQL (8 hours)

Set up a React project, build the main components, and add real-time subscriptions for comments and likes.

3. Authentication (3 hours)

Implement JWT-based authentication in the Go backend and integrate it with the React app.

4. Dockerization (3 hours)

Write Dockerfiles for both the Go backend and React frontend, and create a docker-compose.yml file.

5. Deployment on Azure (4 hours)

Deploy the Docker containers to Azure App Service or AKS.

6. Testing and Final Adjustments (2 hours)

Test API responses, conduct end-to-end testing, and add error handling and UX improvements.

GraphQL Schema for the Go Backend

type Post {
id: ID!
content: String!
author: User!
comments: [Comment!]
likes: Int!
}

type Comment {
id: ID!
content: String!
author: User!
post: Post!
}

type Query {
posts: [Post!]!
}

type Mutation {
createPost(content: String!): Post!
createComment(postId: ID!, content: String!): Comment!
likePost(postId: ID!): Post!
}
