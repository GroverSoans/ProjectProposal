Project Overview

Project Name: MERN Authentication App

Description: This project involves creating a MERN stack application with secure user authentication. Users can register, verify their email addresses, and log in. The backend is built in Node.js with Express, the frontend uses React, and MongoDB serves as the database. The app will be containerized using Docker, and the services will be deployed on Amazon's cloud platform.

Project Architecture

1. Backend (Node.js + Express)

  • Authentication: Implement user registration, login, and email verification using JWT and Nodemailer.
  • Database: Use MongoDB to store user information securely, including hashed passwords.
  • API: Build RESTful APIs to handle authentication and verification processes.

2. Frontend (React)

  • React Interface: Create a clean user interface for registration, login, and account verification.
  • API Integration: Use Axios or Fetch to communicate with the backend APIs.
  • User Experience: Add visual feedback, such as success/failure messages and email verification prompts.

3. Docker Setup

  • Dockerize: Create Dockerfiles for the Node.js backend and React frontend, and use Docker Compose to manage services.

4. Hosting (Amazon)

  • Amazon Hosting: Deploy the frontend to Amazon S3 or CloudFront, and the backend to an Amazon EC2 instance or ECS.

Project Breakdown

1. Backend Development with Node.js (10 hours)

Set up a Node.js project, implement user authentication, and integrate email verification using Nodemailer.

2. Frontend Development with React (8 hours)

Set up a React project, build components for registration and login, and connect them to the backend APIs.

3. Email Verification Integration (3 hours)

Implement email verification in the backend and design the user flow in the React frontend.

4. Dockerization (3 hours)

Write Dockerfiles for the Node.js backend and React frontend, and create a docker-compose.yml file.

5. Deployment on Amazon (4 hours)

Deploy the Docker containers for both services on Amazon S3/CloudFront (frontend) and ECS/EC2 (backend).

6. Testing and Final Adjustments (2 hours)

Test API responses, conduct end-to-end testing, and add error handling and UX improvements.
